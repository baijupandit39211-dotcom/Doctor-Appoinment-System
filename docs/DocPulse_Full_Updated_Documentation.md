# DocPulse Full Updated Documentation

## Product Summary

DocPulse is a full-stack SaaS doctor appointment system for patients, doctors, clinic admins, and super admins.

## Final Stack

- Frontend: Next.js + React + TypeScript + Tailwind CSS
- Backend: Node.js + Express.js
- Database: MongoDB + Mongoose
- Authentication: JWT + HTTP-only Cookies
- UI: shadcn/ui + Radix UI + Lucide React
- Animation: Framer Motion
- Email/Reminder: Nodemailer
- File Upload: Cloudinary

## Core Product Goal

Provide a secure and modern appointment platform where patients can book appointments online and clinics can manage doctors, schedules, appointments, patient records, reminders, and reports.

## Main Deliverables

- Landing page
- Authentication
- Patient dashboard
- Doctor dashboard
- Clinic admin dashboard
- Doctor listing and profile
- Appointment booking and management
- Doctor availability
- Email notifications
- MongoDB database
- Express REST API

## Functional Areas

### Authentication and Access

- Register
- Login
- Logout
- JWT cookie authentication
- Password hashing
- Role-based access

### Patient Experience

- Create profile
- Search doctors
- Filter by specialization
- View doctor profile
- View available time slots
- Book or cancel appointments
- View appointment history

### Doctor Experience

- Manage profile
- Set availability
- View appointments
- Update appointment status
- Review patient basics

### Clinic Admin Experience

- Manage doctors
- Manage departments
- Manage patients
- Manage appointments
- View reports
- Manage clinic settings

### Super Admin Experience

- Manage clinics
- Manage platform users
- Manage subscription plans
- View platform analytics
- Activate or deactivate clinics

## Rules To Keep

- Use Next.js for the frontend.
- Use Node.js and Express.js for the backend.
- Use MongoDB and Mongoose for persistence.
- Use JWT stored in HTTP-only cookies.
- Prevent double booking.
- Keep the system responsive.
- Keep the UI professional and healthcare-focused.
